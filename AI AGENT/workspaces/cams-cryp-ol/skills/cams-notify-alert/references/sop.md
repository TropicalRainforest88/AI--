## 一、商戶通知問題

### 通用通知診斷流程

所有訂單類型（充值、提現、兌換、流動池）共用同一套通知機制。

#### 步驟 1：確認通知狀態

用 OrderID 查詢對應訂單的 `NotifyStatus`：

| NotifyStatus | 說明 | 處理方向 |
|--------------|------|----------|
| 0 (待處理) | 通知尚未發送 | 查通知服務是否正常運作 |
| 1 (成功) | 已成功送達 | 問題在商戶端，確認其接收與處理邏輯 |
| 2 (重試達上限) | 多次重試仍失敗 | 商戶端問題，確認 URL 與網路可達性 |
| 3 (重試中) | 正在重試 | 等待重試結果 |
| 4 (不需要通知) | 未設定通知 URL | 確認商戶通知 URL 設定 |

#### 步驟 2：查詢重試次數

- 查 `NotifyCount` 確認已重試幾次
- 確認 `NotifyAt` 最後一次嘗試時間

#### 步驟 3：確認商戶通知 URL

| 訂單類型 | 通知 URL 欄位 |
|----------|--------------|
| 充值 | Merchant.NotifyDepositURL |
| 提現 | Merchant.NotifyWithdrawURL |
| 兌換 | Merchant.NotifySwapURL |
| 流動池 | Merchant.NotifyLiquidityPoolURL |
| 交易建立 | Merchant.NotifyMapURL（非必填） |

#### 步驟 4：排查通知失敗原因

| 失敗原因 | 排查方式 | 處理 |
|----------|----------|------|
| URL 不可達 | 測試 HTTP 連線 | 聯繫商戶修正 URL |
| 商戶返回非 200 | 查 log 中的回應碼 | 聯繫商戶修正接收端 |
| 簽名驗證失敗 | 確認 SecretKey 一致性 | 重新同步 SecretKey |
| Redis 鎖衝突 | 查 Redis 中的通知鎖 | 等待 2 分鐘鎖過期後重試 |
| 通知服務異常 | 查 cams-job log | 確認服務狀態 |

#### 步驟 5：手動重試通知

- 透過後台操作重試通知
- 注意：每筆訂單有 **2 分鐘 Redis 鎖**，鎖期間無法重複發送
- 確認問題已修正後再重試，避免浪費重試次數

## 二、監控告警處理

### 告警類型與處理方式

#### 1. 鏈高度異常

**觸發條件**：區塊鏈節點同步高度落後或停滯

**處理步驟**：
1. 查 ELK 中鏈節點同步狀態
2. 確認 cryp-admin 回報的節點列表狀態
3. 若節點落後 → 可能是節點服務異常，通知基礎設施團隊
4. 若所有節點都落後 → 可能是鏈本身擁堵

#### 2. 大額轉帳

**觸發條件**：單筆轉帳金額超過設定閾值

**處理步驟**：
1. 用 TransferID 查轉帳詳情
2. 確認轉帳來源與目的
3. 確認是否為正常業務操作（大額提現、錢包轉帳等）
4. 若為異常轉帳 → 啟動風控調查

#### 3. 風險地址充值

**觸發條件**：偵測到來自風險地址的充值交易

**處理步驟**：
1. 用 FromAddress 查風險偵測結果
2. 確認風險來源（TronScan / OKX）和風險標籤
3. 依公司風控政策決定：
   - 標記並觀察
   - 啟動退還流程
   - 凍結相關帳戶
4. 詳細處理見 [08-risk-address-handling.md](08-risk-address-handling.md)

#### 4. 錢包餘額閾值

**觸發條件**：錢包餘額超出設定的上下限

**處理步驟**：
1. 查對應錢包的 TokenWallet 餘額
2. 確認閾值設定（Wallet Threshold）
3. 若低於下限：
   - 提現錢包 → 確認下發是否正常
   - 其他錢包 → 評估是否需要補充資金
4. 若超過上限：
   - 用戶錢包 → 確認歸集是否正常
   - 其他錢包 → 評估是否需要轉出資金

#### 5. 錢包異常

**觸發條件**：錢包狀態或行為異常

**處理步驟**：
1. 查錢包詳情與最近交易記錄
2. 確認是否有未授權的交易
3. 必要時暫停該錢包的操作

#### 6. 提現時長異常

**觸發條件**：提現訂單處理時間超過預期

**處理步驟**：
1. 用 OrderID 查提現流程各階段耗時
2. 確認卡在哪個環節
3. 按 [02-withdraw-issues.md](02-withdraw-issues.md) 排查

#### 7. 淨資產差異

**觸發條件**：帳面記錄與鏈上實際數據不一致

**處理步驟**：
1. 比對帳面 HoldingAmount 與鏈上實際餘額
2. 查最近的 Transfer 記錄是否有遺漏
3. 確認是否有未入帳的鏈上交易
4. 若差異持續 → 需進行完整對帳 (Reconciliation)
5. 確認每日資產快照 (SnapWalletAsset) 是否正常建立
