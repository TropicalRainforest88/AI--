# 通知與告警排查

## 商戶通知 (Notify)

所有訂單類型共用通知機制：

1. 確認商戶對應的 Notify URL 是否有設定
2. 查 NotifyStatus：
   - 0 待處理 → 通知尚未發送
   - 1 成功
   - 2 重試達上限 → 商戶端可能有問題
   - 3 重試中
   - 4 不需要通知
3. 查 NotifyCount 確認重試次數
4. 確認是否有 Redis 鎖衝突（2 分鐘鎖）

## 監控告警 (Monitor Alert)

| 告警類型 | 排查方向 |
|----------|----------|
| 鏈高度異常 | 查 ELK 中鏈節點同步狀態 |
| 大額轉帳 | 用 TransferID 查轉帳詳情 |
| 風險地址充值 | 用 FromAddress 查風險偵測結果 |
| 錢包餘額閾值 | 查對應錢包的 TokenWallet 餘額 |
| 提現時長異常 | 用 OrderID 查提現流程各階段耗時 |
| 淨資產差異 | 比對帳面記錄與鏈上實際數據 |
