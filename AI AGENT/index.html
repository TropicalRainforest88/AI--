PR 生命週期

1
開發完成 — Claude Code 執行 /push-pr，建分支、commit、開 PR
2
Code Review — 自動觸發 codereview agent 四面向審查
3
審查結果 — approved 則 Telegram 提醒部署；need-change 則修改後重推
4
部署 — 告訴 Agent「merge 並部署 PR #N」，觸發 deploy workflow
5
驗證 — 執行 /verify-pr #N，SSH 逐項驗證，標記 verified

Label 狀態機

Label含義觸發
review等待 code review自動
need-change需修改agent
approved審查通過agent
deployed已部署workflow
verified驗證完成skill
